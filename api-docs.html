🎵 Plum API Documentation

Artist Booking Platform - Complete API Reference

Authentication

POST /api/auth/register

Register a new user account

{ "email": "user@example.com", "password": "password123", "full_name": "John Doe", "user_type": "client" // or "artist" }
Required Fields: email, password, full_name, user_type
Response: 201 Created with user_id
POST /api/auth/login

Login user and create session

{ "email": "user@example.com", "password": "password123" }
Response: User object with authentication info
POST /api/auth/logout

Logout current user and destroy session

Auth Required: YES

Artists

GET /api/artists

List all verified artists with pagination

// Query Parameters: ?page=1&limit=12 // Response includes pagination info and artist cards
GET /api/artists/{id}

Get complete artist profile with services and reviews

Response includes: Profile details, services, recent reviews
GET /api/artists/search

Search and filter artists

// Query Parameters: ?q=jazz&category=musician&location=new%20york&min_rating=4&sort=rating
POST /api/artists/profile

Update artist profile (artists only)

Auth Required: YES (Artist account)

Bookings

GET /api/bookings

List user's bookings (client or artist view)

// Query Parameters: ?status=confirmed // Filter by status // Returns different data based on user_type
Auth Required: YES
GET /api/bookings/{id}

Get booking details

Auth Required: YES
Only client or artist in booking can access
POST /api/bookings

Create new booking

{ "artist_id": 123, "event_date": "2026-04-15", "event_type": "Wedding", "event_location": "Manhattan", "total_amount": 3800, "deposit_amount": 950, "notes": "Please arrange outdoor setup" }
Auth Required: YES (Client account)
PUT /api/bookings/{id}

Update booking status or payment status

{ "booking_status": "confirmed", // or "completed", "cancelled" "payment_status": "paid" // or "pending", "refunded" }
Auth Required: YES

Reviews

GET /api/reviews/artist/{id}

Get reviews for an artist with stats

// Query Parameters: ?page=1&limit=10&sort=recent // or helpful, rating
POST /api/reviews

Create review for completed booking

{ "booking_id": 456, "artist_id": 123, "rating": 5, "comment": "Absolutely amazing performance! Would book again." }
Auth Required: YES (Client account)
Booking must be marked as "completed"

HTTP Response Codes

  • 200 - OK: Request successful
  • 201 - Created: Resource created successfully
  • 400 - Bad Request: Invalid parameters or missing fields
  • 401 - Unauthorized: Authentication required
  • 403 - Forbidden: Access denied (wrong role)
  • 404 - Not Found: Resource doesn't exist
  • 405 - Method Not Allowed: Wrong HTTP method
  • 500 - Server Error: Internal error
